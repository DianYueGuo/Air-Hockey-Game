# E2

What changed
- Added calibration persistence to an OS-appropriate user data directory.
- Play screen loads calibration data and uses it to map camera coordinates to mallet positions.
- Calibration screen saves data after each capture step.

Manual test steps
- Run `python -m air_hockey.main` and complete Calibration steps.
- Restart the app and verify mallet mapping reflects the saved calibration.

Known issues
- Calibration load does not currently notify if data is missing or invalid.
